Installation

On Windows

Note: I would never recommend trying docker on windows.You might doubt your existence and feel like pulling all the hair from your head.

Note:Ubuntu WSL is not sufficient to run docker.Shall not be considered a substitute.

For Windows Home:

  • : As Docker is only available for Windows enterprise edition
  • : After Installing docker
  • : Edit Windows Version in Registry
  • : Press Windows + R and write regedit
  • : In the Registry Editor, go to \HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion
  • : Right-click on EditionID and Click Modify
  • : Change Value Data to "Professional"
  • : Press OK
  • Install Docker Desktop
  • After the installation, you can change the EditionID back to "Core"
  • After this install docker toolbox
  • Ensure that nothing like Hyper-V or Windows Virtualization platform is Running
  • : Check this thread

Commands

  • : To start : net start com.docker.service
  • : docker ps -a : List all the containers
  • : docker run : Run an image.If it does not exist it will be pulled from docker-hub
  • : docker stop ID : Stop a running container
  • : docker rm ID(First few letters will do) : Remove a container permanently
  • : docker images : List of images on System
  • : docker rmi : Remove an image
  • : docker pull : Pull the image
  • : docker run -it name: Log into container
  • 0 : docker rm a b c : Remove multiple containers
  • 1 : docker rmi name : Remove docker image
  • 2 : docker exec ID cat /abc : Access files inside docker
  • 3 : docker run --name webapp nginx:1.14-alpine
  • 4 : docker inspect container_ID : Additional details about container
  • 5 : docker attach: Check local output
  • 6 : docker build -t name/tag .
  • 7 : docker run -it tag uilitly
  • 8 : docker push : Add the image to dockerhub
  • 9 : docker compose up : Bring up the application

Commands On Running Container

  • : docker run name/abcd : Run in attached mode
  • : Ctrl+C: Stop the container
  • : docker run -i : Run in interactive mode
  • : docker run -d : Run in background mode/detached mode.
  • : docker run -e :Set an environment variable
  • : Use docker inspect for finding environment variables
  • : Config section has the environment variables
  • : docker run --name NAME

Port Mapping

  • Docker Host/Engine : The host where docker is installed
  • IP on which I can access the application from web browser
  • : Use the default internal IP(Only accessible within the docker host)
  • : Ex- docker run -p 80:5000 abc/xyz
  • : Or map it to a port use external mapping

Volume Mapping

  • Containers live as long as process inside them exists

To ensure a local data store.Map a directory outside the container on the Docker.

  • Ex: docker run -v /opt/datadir: /var/lib/mysql MySQL
  • opt/datadir mapped to /var/lib/mysql
